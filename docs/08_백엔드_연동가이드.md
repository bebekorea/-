# 백엔드 연동 가이드

이 문서는 베베펫 사이트의 두 가지 백엔드 연동(방문예약 이메일, 공지사항 CMS)을 운영자가 직접 세팅할 수 있도록 단계별로 정리한 가이드입니다.

## 1) EmailJS — 방문예약 이메일 자동 발송

방문예약 폼(`방문예약등록` 버튼 → 모달) 제출 시 `bebepet0409@naver.com`으로 신청 내역이 자동 발송됩니다. 클라이언트 사이드 SDK라 서버/API 라우트가 필요 없습니다.

### 셋업 단계

1. **계정 생성** — [https://www.emailjs.com](https://www.emailjs.com) 가입 (Google 계정으로 간편 가입 가능). 무료 플랜이 월 200건까지 발송.

2. **이메일 서비스 등록**
   - 대시보드 → `Email Services` → `Add New Service`
   - Naver 사용 시 SMTP 선택 → bebepet0409@naver.com 계정의 앱 비밀번호 입력 (Naver 계정 설정 → 보안 → 앱 비밀번호 발급)
   - Gmail/Outlook도 동일 방식 가능
   - 등록 후 생성된 **Service ID** 복사 (예: `service_abc1234`)

3. **이메일 템플릿 작성**
   - 대시보드 → `Email Templates` → `Create New Template`
   - To Email: `bebepet0409@naver.com`
   - Subject: `[베베펫] 방문예약 신청 — {{name}}`
   - Content:
     ```
     이름: {{name}}
     전화: {{tel}}
     지역: {{sido}}

     상담 목적:
     {{subject}}
     ```
   - 저장 후 **Template ID** 복사 (예: `template_xyz5678`)

4. **Public Key 확인**
   - 대시보드 → `Account` → `General` → `Public Key` 복사 (예: `abcdEFGH1234`)

5. **환경변수 설정**
   - 프로젝트 루트의 `.env.local.example`을 `.env.local`로 복사
   - 다음 3개 값을 채움:
     ```
     NEXT_PUBLIC_EMAILJS_SERVICE_ID=service_abc1234
     NEXT_PUBLIC_EMAILJS_TEMPLATE_ID=template_xyz5678
     NEXT_PUBLIC_EMAILJS_PUBLIC_KEY=abcdEFGH1234
     ```
   - 개발 서버 재시작 (`npm run dev` 중단 후 다시 실행)
   - 배포 환경(Vercel 등)에서도 동일 환경변수 등록 필요

### 테스트

- 사이트의 `방문예약등록` 버튼 클릭 → 폼 작성 → 제출
- 정상 시: 모달이 닫히고 alert "예약이 접수되었습니다" 노출
- bebepet0409@naver.com 받은편지함 확인
- 실패 시: 브라우저 콘솔에서 `[RegisterModal] EmailJS send failed` 로그 확인

### 발송량/요금

- 무료: 월 200건
- $7/월: 1000건
- 단순 폼 한 개라면 무료로 충분

## 2) Notion CMS — 공지사항 페이지 운영

`/news` 페이지의 기사 목록을 Notion 데이터베이스로 관리합니다. 운영자가 Notion에서 새 글을 추가하면 최대 60초 후 사이트에 자동 반영됩니다.

### 셋업 단계

1. **Notion Integration 생성**
   - [https://www.notion.so/my-integrations](https://www.notion.so/my-integrations) 접속
   - `+ New integration` 클릭
   - 이름: `베베펫 사이트` (자유), 워크스페이스: 베베펫 워크스페이스 선택
   - 권한: Read content만 체크하면 충분
   - 저장 → **Internal Integration Token** 복사 (예: `secret_xxxxxxxxx`)

2. **공지사항 데이터베이스 생성**

   Notion 워크스페이스에서 새 페이지 생성 → `/database - Full page` 입력 → 다음 컬럼 추가 (**컬럼명은 영문 정확히 일치해야 함**):

   | 컬럼명 | 타입 | 설명 |
   |---|---|---|
   | `Title` | Title | 기사 제목 (한글) |
   | `Date` | Date | 발행 날짜 |
   | `Source` | Text | 출처 (예: "한국경제") |
   | `URL` | URL | 원문 링크 (선택) |
   | `Published` | Checkbox | 발행 여부 (체크 시에만 사이트에 노출) |

3. **DB와 Integration 연결**
   - 생성한 데이터베이스 페이지 우상단 `⋯` → `Add connections` → 위에서 만든 Integration 선택
   - 연결 안 하면 API가 DB를 못 읽음 (Notion 권한 모델)

4. **데이터베이스 ID 추출**
   - 데이터베이스 페이지를 브라우저에서 열기
   - 주소창의 URL: `https://www.notion.so/{workspace}/{이부분이_DB_ID}?v=...`
   - DB_ID는 32자리 영숫자 (예: `8a3c2f1...d9e0b1`)

5. **환경변수 설정**
   - `.env.local`에 추가:
     ```
     NOTION_TOKEN=secret_xxxxxxxxx
     NOTION_NEWS_DB_ID=8a3c2f1d9e0b1...
     ```
   - 개발 서버 재시작
   - 배포 환경에서도 동일 환경변수 등록

### 운영

- Notion DB에 새 행 추가 → `Title`, `Date`, `Source` 입력 → `Published` 체크
- 최대 60초 후 `/news` 페이지에 자동 노출 (날짜 내림차순)
- 임시 비공개로 돌리려면 `Published` 체크 해제
- 영문 사이트(English)는 별도 컬럼 안 만들었으므로 한글 그대로 노출. 영문 분리 필요 시 `TitleEn`, `SourceEn` 컬럼 추가 후 `src/lib/notion.ts` 수정 필요

### Fallback 동작

- 환경변수가 비어 있거나 Notion 호출이 실패하면 → 코드 안에 박아둔 fallback 기사 6건이 노출됨 (`src/app/news/page.tsx`의 `FALLBACK_ARTICLES`)
- Notion이 정상 연결됐는데 DB가 비어 있으면 → "공지가 없습니다" 빈 상태 노출

### 글 추가/삭제

- **추가**: Notion DB에 새 행 입력 → `Published` 체크 → 60초 후 사이트 노출
- **삭제(소프트)**: `Published` 체크 해제 → 사이트에서 즉시 사라짐 (DB엔 보관)
- **삭제(완전)**: Notion 행 자체를 삭제 → DB에서도 사라짐
- **수정**: 행의 컬럼 값 수정 → 60초 후 반영

## 3) Notion Assets — 로고/사진/영상 교체 시스템

운영자가 사이트의 시각 자산(로고, 카테고리 사진 6장, 조직도, Hero 영상, Contact 배경)을 Notion DB에서 URL만 바꿔 교체할 수 있습니다. 이미지/영상 파일 자체는 외부 호스팅(Imgur/Cloudinary/Imgbb 등 무료 서비스)에 올리고, 그 직접 링크를 Notion에 입력합니다.

### 셋업 단계

1. **Assets DB 생성**
   - Notion에서 새 데이터베이스 페이지 생성 (이름 자유, 예: "BebePet Assets")
   - 컬럼 (영문 정확히):

     | 컬럼명 | 타입 | 설명 |
     |---|---|---|
     | `Key` | Title | 자산 식별자 (아래 표 참조) |
     | `URL` | URL | 이미지/영상 직접 링크 |
     | `Active` | Checkbox | 체크 시 사이트 반영 |
     | `Notes` | Text | 운영자 메모 (선택) |

2. **DB와 Integration 연결**
   - DB 페이지 ⋯ → `Add connections` → 공지사항용으로 만든 Integration 동일 사용

3. **DB ID 추출 후 .env.local 추가**
   ```
   NOTION_ASSETS_DB_ID=xxxxxxxxxxxx
   ```

4. **사이트 재배포** (Vercel 등)

### 사용 가능한 Key 목록

| Key | 위치 | 설명 |
|---|---|---|
| `logo` | 헤더 + 푸터 | "BEBE PET" 워드마크 자리. 이미지 URL 등록 시 텍스트 대신 이미지 노출. 로고 PNG/SVG 권장 (배경 투명) |
| `category_adopt` | 입양 섹션 | 메인 사진 |
| `category_location` | 병원 섹션 | 메인 사진 |
| `category_product` | 용품 섹션 | 메인 사진 |
| `category_beauty` | 미용 섹션 | 메인 사진 |
| `category_partners` | 사료/간식 섹션 | 메인 사진 |
| `category_global` | 글로벌 섹션 | 메인 사진 |
| `org_chart` | TRUST 섹션 (PC + 모바일) | 조직도 이미지. 등록 시 placeholder 박스 대신 이미지 노출 |
| `hero_video` | Hero stage 3 (PC) | brand.mp4 대체 영상 URL |
| `contact_bg` | Contact 섹션 (PC) | 배경 사진 |

### 운영자 흐름

1. **이미지를 외부 호스팅에 업로드**
   - 무료 옵션:
     - [Imgur](https://imgur.com) — 가입 없이 업로드 가능, 직접 링크 제공
     - [ImgBB](https://imgbb.com) — 가입 후 무료 무제한
     - [Cloudinary](https://cloudinary.com) — 25GB 무료, 가입 필요
   - 업로드 후 **"Direct Link"** 또는 **"Image URL"** 복사 (`.jpg`/`.png`/`.webp`로 끝나는 URL)

2. **Notion DB에 입력**
   - 해당 Key 행이 없으면 새 행 추가, 있으면 URL 컬럼만 수정
   - `Active` 체크
   - 저장

3. **확인**
   - 사이트에서 최대 60초 후 새 이미지로 자동 교체
   - 캐시 강제 갱신 원하면 시크릿 모드 새로고침

### 영상 (hero_video) 주의사항

- 영상 파일은 용량이 크므로 Imgur는 부적합
- Cloudinary나 자체 CDN, 또는 Vercel 정적 호스팅 권장
- `.mp4` 형식, 권장 길이 5-15초 루프, 해상도 1920×1080 이하
- 영상 URL은 `https://...mp4`로 끝나야 함

### Fallback 동작

- 환경변수가 비었거나 Notion 호출 실패 시 → 코드의 기본 자산 (Unsplash 사진, /videos/brand.mp4, /images/contact-bg.jpg) 사용
- Notion에 Key가 있지만 `Active`가 false면 → 기본 자산 사용
- 즉, 운영자가 잘못 등록해도 사이트가 깨지지 않음

## 보안 주의사항

- `.env.local`은 **git에 절대 커밋하지 말 것** (`.gitignore`에 `.env*`이 이미 등록됨)
- EmailJS의 Public Key는 클라이언트에 노출돼도 안전한 키지만, Service/Template ID와 함께 사용되므로 도메인 화이트리스트를 EmailJS 대시보드에서 설정해두는 게 좋음 (Account → General → Allowed Origins)
- Notion Token은 서버에서만 사용 (NEXT_PUBLIC_ 접두사 안 붙임 → 클라이언트 번들에 안 들어감)

## 비용

- 둘 다 **현재 사이트 규모에서는 완전 무료**
- EmailJS: 월 200건 무료 (예약 폼이 월 200건 넘으면 $7/월)
- Notion API: 호출 횟수 무제한 (Notion 워크스페이스 계정만 있으면 됨)
