## 1. 인트로 (ScrollHero) 시퀀스

### 1.1 자동 진행 페이즈

| 페이즈 | 시간 | 가시 요소 |
|---|---|---|
| boot | 300ms | 검은 화면 |
| line | 1900ms | 한 글자씩 타이핑 (intro.typing 키), 커서 깜빡 |
| copy | 700ms + 500ms hold | 타이핑 완료 + 커서 유지 |
| fadeOut | 650ms | 텍스트 페이드 아웃 |
| banner | 1100ms | 가로 28vh 띠로 비디오 등장 |
| full | 1000ms | 비디오 풀스크린 모핑 |
| done | — | stage 1 인터랙션 가능 상태 |

### 1.2 Skip 버튼

- **위치**: 타이핑 텍스트 우측 14.5vw 떨어진 곳
- **표시 조건**: phase ∈ {line, copy} 동안만
- **클릭 동작**:
  1. 모든 인트로 타이머 즉시 클리어
  2. `skipped = true` (200ms 동안 유지)
  3. phase = `done`, stage = 1
  4. 비디오 프레임 morph / 비디오 페이드 모두 0ms (즉시 풀스크린)
  5. 타이핑 텍스트 즉시 사라짐 (가시 1프레임 잔상 0)

### 1.3 Stage 전환 (휠/터치)

`done` 페이즈 진입 후만 활성:
- **휠 다운** (`deltaY > 5`): stage++ (max 3)
- **휠 업**: stage-- (min 1)
- **stage 3에서 추가 다운**: 페이지 레벨 점프 → CategoryIndex (#index)
- **stage 1에서 추가 업**: 페이지 레벨 점프는 hero가 첫 섹션이라 없음

각 stage 전환은 1100ms `cubic-bezier(0.65, 0, 0.35, 1)` 모핑.

### 1.4 Reset 동작

`heroResetTick`이 증가할 때:
- **트리거**: 플로팅 백투탑 버튼 클릭 OR 다른 섹션에서 wheel-up으로 hero 복귀
- **결과**: stage = 1로 리셋 (사용자가 항상 "평생 함께할 첫만남" 첫 컴포지션을 다시 보게 됨)

## 2. CategoryIndex Hover Preview

### 2.1 Hover 진입

- **트리거**: 5개 마퀴 라벨 중 하나에 마우스 오버
- **상태**: `setHovered("{id}|0")`
- **시각 효과**:
  1. `PreviewLayer` opacity 0→1 (즉시)
  2. 자식 `CategoryHoverPreview` 마운트 → 5단계 staggered 페이드 인:
     - visual: `translateX(±32px) scale(0.985)` → 0/1, 520ms
     - eyebrow label: `translateY(18px)` → 0, 520ms, 110ms 지연
     - caption: ↑ 220ms 지연
     - body: ↑ 330ms 지연
     - CTA / logoFlow: ↑ 440ms 지연

### 2.2 Hover 지속성

- **다른 카테고리 hover 전까지 유지** (사용자 요청 사항)
- 섹션을 떠나도 preview는 그대로, 다시 #index에 들어오면 같은 preview 보임

### 2.3 스크롤 중 Hover 잠금

- 페이지 휠/터치 이벤트 발생 시 `lastScrollRef = Date.now()`
- 마지막 스크롤로부터 1500ms 이내 hover 이벤트는 **무시** → 스크롤 중 카테고리 사이를 지나가도 preview가 휙휙 바뀌지 않음

### 2.4 Top Nav 라우팅

- 분양 / 병원 / 사료&간식 / 용품 / 미용 클릭 시:
  1. `scrollToSection({id})` → #index로 점프 스크롤
  2. `preselectedCategory = id`, `preselectTick++` 증가
  3. CategoryIndex의 useEffect가 tick 감지 → `setHovered(id)` → 해당 preview 자동 표시
- 같은 항목을 다시 클릭해도 tick 카운터가 올라가서 다시 활성화됨

## 3. RegisterModal (방문 예약)

### 3.1 오픈/닫기

- **오픈**: Header `방문예약등록` / Hero stage 1 RESERVATION (모바일) / BEAUTY&SPA preview의 `예약하기` 버튼 클릭
- **닫기**: 백드롭 클릭 / 우상단 X 버튼 / Esc 키

### 3.2 슬라이드 애니메이션

- **2-단계 마운트**: 
  1. `mounted = true` → DOM에 panel 추가 (translateX 100% off-screen)
  2. 2× rAF 후 `visible = true` → translateX 0 (1100ms 슬라이드 인)
- **닫힘**: `visible = false` → 슬라이드 아웃 → 1100ms 후 `mounted = false`

### 3.3 폼 검증

| 필드 | 검증 |
|---|---|
| 이름 | required |
| 전화번호 (010) | maxLength 3, numeric |
| 전화번호 (4자리 × 2) | maxLength 4, numeric |
| 지역 시/도 | required |
| 상담목적 (radio) | optional (선택 안 해도 제출 가능, 향후 변경 검토) |
| 개인정보 동의 | required (checkbox) |

### 3.4 제출 동작 (현재)

```
alert(t("reg.success"))  // "방문 예약이 접수되었습니다. (데모)"
onClose()
```

## 4. 다국어 (i18n)

### 4.1 토글

- **위치**: Header 우측 `KOR | ENG`
- **저장**: `localStorage["bebepet-lang"]`
- **즉시 반영**: `useSyncExternalStore`로 모든 컴포넌트 자동 리렌더
- **html lang 속성** 자동 동기화 (a11y / SEO)

### 4.2 키 검색 패턴

`src/lib/i18n.tsx`의 `ko` / `en` 사전 객체에서 동일 키로 양방향 매핑.

```tsx
const { t } = useLang();
t("nav.adopt")  // "분양" / "ADOPT"
```

배열 값은 `ta(key)`로 받아 `.map()` 가능.

## 5. 카카오맵 (LocationMap)

- **소스**: `https://staticmap.kakao.com/map/mapservice` 정적 이미지 API (무료, API 키 불필요)
- **좌표**: WCONGNAMUL 투영 시스템 (MX=530693, MY=926031)
- **줌**: urlLevel=2
- **해상도**: 1024×720 SCALE=2 (retina 2048×1440)
- **인터랙션 X**: 정적 이미지라 줌/드래그 불가. 클릭하면 `https://map.kakao.com/?...` 풀맵으로 새 탭 오픈
- **마커**: 카카오 정적맵에는 핀이 없어 absolute 오버레이로 직접 그림 (빨간 드롭 + 흰 점 + drop shadow + "베베펫 본사" 흰 pill 라벨)

## 6. 헤더 테마 자동 전환

`src/app/page.tsx`의 `headerTheme` 계산식:

```
if (hero stage 1) → 'dark' (흰 글자)
elif (hero 다른 stage) → 'light' (검은 글자)
elif (index + activePreview) → preview의 bg에 맞춰 light/dark
elif (DARK_SECTIONS에 포함) → 'dark'
else → 'light'
```

`DARK_SECTIONS = { hero, index, contact }` (BEAUTY는 light로 변경됨, 카드 bg가 white라서)

## 8. 키보드 / 접근성

- **Esc**: RegisterModal 닫기
- **Tab**: 모든 폼 인풋 / 버튼 / 링크 포커스 가능
- **aria-label**: 아이콘 버튼 (햄버거, 백투탑, IG, 카카오맵 등) 모두 부착
- **alt 속성**: 이미지 alt 부착 (장식용은 `aria-hidden="true"`)
- **포커스 outline**: Tailwind 기본 사용
