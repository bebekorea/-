# 05. 콘텐츠 운영 매뉴얼

본 문서는 베베펫 웹사이트의 콘텐츠 — 텍스트 · 이미지 · 매장 정보 · 사업자 정보 등 — 를 운영자가 직접 수정·반영하기 위한 절차를 정의합니다.

---

## 1. 작업 환경

### 1.1 필요 자원

| 항목 | 내용 |
|---|---|
| 브라우저 | Chrome 최신 버전 권장 |
| 계정 | GitHub 계정 (https://github.com/join, 무료 가입) |
| 권한 | 본 프로젝트 GitHub 리포지토리에 대한 Write 권한 (에이전시가 초대 발송) |

### 1.2 변경 반영 흐름

GitHub에서의 모든 커밋은 호스팅 서비스(Vercel)의 자동 배포 파이프라인을 통해 약 5분 이내 운영 사이트에 반영됩니다.

> 변경 사항은 즉시 운영 환경에 반영되므로, 신중히 작업하시고 가능한 경우 영업 외 시간에 진행을 권장합니다.

## 2. 매장 기본 정보 수정

매장 주소 · 영업시간은 사이트 내 Visit(`#contact`) 섹션에 표시됩니다.

### 수정 위치
`src/lib/i18n.tsx`

### 절차

1. GitHub 리포지토리에서 `src/lib/i18n.tsx` 파일 진입
2. 우측 상단의 **편집(Edit, ✏️ 아이콘)** 클릭
3. 다음 항목을 찾아 따옴표 안의 **값**만 수정

```ts
"contact.value.loc": "충청남도 천안시 서북구 두정동 864",
"contact.value.period": "매일 10:00 ~ 20:00 (연중무휴)",
```

4. 영문 사전(`const en: Dict = {`) 영역에서 동일 키를 찾아 영문 값도 함께 갱신

```ts
"contact.value.loc": "864 Dujeong-dong, Seobuk-gu, Cheonan, Chungnam",
"contact.value.period": "Daily 10:00 – 20:00 (open year-round)",
```

5. 페이지 하단의 **Commit changes** 클릭 → 변경 내용을 한 줄로 기재 (예: "주소 변경") → 커밋

### 작업 시 유의사항

- 키(좌측, 예: `"contact.value.loc"`)는 변경 금지. 값(따옴표 안의 우측)만 수정.
- 줄 끝의 콤마(`,`)와 따옴표(`"`)는 그대로 유지.
- 한국어 사전과 영문 사전을 함께 수정해야 양쪽 언어 모두 반영됩니다.

> 전화번호 · 이메일은 현재 사이트 콘텐츠 영역에 노출되지 않습니다.

## 5. 이미지 교체

이미지는 위치에 따라 두 유형으로 분류되며, 교체 절차가 상이합니다.

### 5.1 프로젝트 내 이미지 (`public/images/`)

대상: 파트너 로고 8종(`partner-name1.png` ~ `partner-name8.png`) 등.

#### 신규 업로드

1. GitHub 리포지토리에서 `public/images/` 폴더 진입
2. 우측 상단 **Add file → Upload files** 클릭
3. 파일 드래그 앤 드롭
4. 페이지 하단 **Commit changes** 클릭

#### 기존 파일 교체

1. 교체 대상 파일을 휴지통 아이콘으로 삭제
2. 동일 파일명으로 재업로드

> 기존 파일과 동일한 파일명을 유지해야 별도 코드 수정 없이 사이트에 반영됩니다.

### 5.2 외부 URL 기반 이미지 (Unsplash 등)

대상: ADOPT 종 사진 3장, PRODUCT 카드 6장.

#### 수정 위치
`src/components/sections/CategoryIndex.tsx`

#### 대상 영역

```tsx
const PRODUCT_CARDS: Array<{ labelKey: string; photo: string }> = [
  { labelKey: "product.card.care", photo: "https://images.unsplash.com/photo-XXXX?..." },
];

const ADOPT_SPECIES: Array<{ labelKey: string; photo: string }> = [
  { labelKey: "adopt.species.large", photo: "https://images.unsplash.com/photo-YYYY?..." },
];
```

#### 옵션 A. 다른 외부 URL로 교체

1. https://unsplash.com 또는 자체 CDN에서 이미지 URL 확보
2. 위 코드의 `photo: "..."` 따옴표 안 URL을 신규 URL로 교체

#### 옵션 B. 자체 촬영 이미지로 교체 (권장)

1. `public/images/` 폴더에 신규 이미지 업로드 (예: `adopt-large-dog.jpg`)
2. 코드 내 URL을 `"/images/adopt-large-dog.jpg"` 형식으로 교체

```tsx
{ labelKey: "adopt.species.large", photo: "/images/adopt-large-dog.jpg" },
```

자체 이미지 사용 시 광고 신뢰성과 운영 비용 양 측면에서 유리합니다.

### 5.3 Hero 영상 (`vis.mp4`, `brand.mp4`)

`public/videos/` 폴더에 동일 파일명으로 신규 영상을 재업로드합니다.

권장 사양:
- 해상도 1920×1080
- 인코딩 mp4 (H.264)
- 길이 10–30초, 무한 루프 가능
- 파일 크기 30MB 이하

## 7. 매장 위치 (카카오맵)

### 수정 위치
`src/components/sections/LocationMap.tsx`

### 좌표 갱신 절차

매장 이전 또는 좌표 정정 시:

1. https://map.kakao.com 접속 후 신규 주소 검색
2. 매장 위치 클릭으로 정보창 열기
3. 정보창 우측 **공유 ↗** 클릭 → **지도 첨부** 탭 선택
4. 표시된 HTML 코드에서 `urlX=...`, `urlY=...`, `MX=...`, `MY=...` 값을 복사
5. `LocationMap.tsx`의 다음 두 URL의 동일 위치에 값을 반영

```tsx
const KAKAO_LARGE_VIEW_URL =
  "https://map.kakao.com/?urlX=...&urlY=...&urlLevel=2&map_type=TYPE_MAP&map_hybrid=false";
const KAKAO_STATIC_IMG =
  "https://staticmap.kakao.com/map/mapservice?...&MX=...&MY=...&...";
```

### 마커 라벨 변경

```tsx
<text fontSize="13" letterSpacing="0.22em" fontWeight="500">
  베베펫 본사
</text>
```

`>`와 `<` 사이의 텍스트만 수정합니다 (예: "베베펫 천안점").

## 11. FAQ

**Q. 한국어만 수정하고 영문은 미변경 시 어떻게 표시되는지?**
A. 한국어 모드에서만 신규 값으로 표시되며, 영문 모드에서는 변경 전 값이 노출됩니다. 두 사전을 함께 수정해야 양쪽 언어 모두 반영됩니다.

**Q. 잘못 수정한 경우의 복구 절차는?**
A. GitHub의 해당 파일 페이지에서 우상단 **History** → 이전 커밋 선택 → 우측 ⋯ → **Revert** 메뉴로 되돌립니다.

**Q. 변경이 사이트에 반영되지 않는 경우의 점검 사항은?**
A. (1) 약 5–10분 대기 후 브라우저 강력 새로고침(`Ctrl+Shift+R` / Mac은 `⌘+Shift+R`), (2) Vercel 대시보드에서 빌드 상태 확인, (3) 빌드 실패 시 에이전시에 빌드 로그와 함께 문의.

**Q. 메뉴 항목 추가·삭제 또는 신규 페이지 추가는 운영자가 직접 가능한지?**
A. 메뉴 구조 변경 및 신규 라우트 추가는 다수의 코드 파일 변경이 수반되는 개발 작업입니다. 별도 의뢰로 진행하시기 바랍니다.

## 12. 작업 가이드라인

| 권장 사항 | 금지 사항 |
|---|---|
| 변경 전 GitHub History에서 현재 상태 확인 | 키 이름(예: `"footer.site.value"`) 변경 |
| 작은 단위로 점진적 변경 후 운영 환경 확인 | 따옴표(`"`) · 콤마(`,`) 임의 삭제 |
| 커밋 메시지에 변경 내용 명확히 기재 | `package.json`, `next.config.ts`, `tsconfig.json` 등 설정 파일 직접 수정 |
