## 3. 외부 서비스 계정

| 서비스 | 용도 | 계정 명의 | 비용 | 비고 |
|---|---|---|---|---|
| 카카오맵 정적 이미지 API | Visit 섹션 지도 | 계정 불필요 | 무료 | Public API, 키 없음 |
| Unsplash CDN | 카테고리 이미지 (placeholder) | 계정 불필요 | 무료 | 운영 전 자체 이미지로 교체 권장 |
| Google Fonts (Jost, Noto Serif KR) | 영문/한문 serif 폰트 | 계정 불필요 | 무료 | OFL |
| Pretendard CDN (jsdelivr) | 한국어 본문 폰트 | 계정 불필요 | 무료 | OFL |
| GitHub | 코드 저장 | 발주처 명의 | 무료 (private 리포 포함) | Write 권한 운영자에게 부여 |
| Vercel | 호스팅 | 발주처 명의 | 무료 티어 한도 내 | 월 100GB 대역 |

발주처 명의 계정이 필요한 항목은 GitHub와 Vercel 두 가지입니다. 이메일 / 패스워드 / 2FA 정보는 별도 보안 채널로 전달하며 본 문서에는 기재하지 않습니다.

## 4. 데이터 저장소

본 프로젝트는 정적 사이트로 데이터베이스를 사용하지 않습니다. 방문 예약 폼은 클라이언트 측 검증 후 alert로 응답하는 상태이며 외부로 데이터가 전송되지 않습니다.

## 5. 권한 인계 절차

### 6.1 GitHub

1. 에이전시 개발자가 발주처 GitHub 계정을 리포지토리 **Owner** 또는 **Admin**으로 초대
2. 발주처가 초대 수락 → 권한 이전
3. 에이전시 개발자 권한은 Maintainer 또는 Collaborator로 단계 강등 (유지보수 기간 동안)
4. 유지보수 종료 후 에이전시 권한 제거

### 6.2 Vercel

1. 에이전시가 Vercel 프로젝트의 **Transfer to another team** 메뉴로 발주처 팀에 이전
2. 또는 발주처가 동일 GitHub 리포를 발주처 명의 Vercel 계정에 신규 import
3. 도메인 / 환경변수 / 빌드 설정 검수 후 라이브 전환

### 6.3 도메인

도메인 등록 기관 콘솔에서 Vercel이 안내하는 A 레코드 또는 CNAME으로 DNS 설정. 등록 기관 계정은 발주처가 직접 보유 · 관리.
