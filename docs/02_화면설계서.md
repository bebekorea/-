# 02. 화면 설계서

각 라우트 / 섹션별 컴포넌트 트리와 레이아웃 정의.

---

## 1. 라우트 맵

| 경로 | 컴포넌트 | 비고 |
|---|---|---|
| `/` | `src/app/page.tsx` | 단일 페이지 — 4 섹션 풀스크린 스냅 |
| `/news` | `src/app/news/page.tsx` | 공지사항 목록 |
| `/privacy` | `src/app/privacy/page.tsx` | 개인정보처리방침 (정적) |

---

## 2. `/` 홈 — 풀페이지 스냅 구조

```
┌──────────────────────────────────────────┐
│ Header (fixed, transparent, theme adapts)│ ← 모든 섹션 위에 떠있음
├──────────────────────────────────────────┤
│                                          │
│  ScrollHero  (#hero, 100vh)              │ ← 인트로 → stage 1/2/3
│                                          │
├──────────────────────────────────────────┤
│  CategoryIndex  (#index, 100vh)          │ ← 카테고리 마퀴 + hover preview
├──────────────────────────────────────────┤
│  Contact  (#contact, 100vh) │ ← VISIT 섹션
├──────────────────────────────────────────┤
│  Footer  (auto height)                   │ ← 브랜드 + 사업자정보
└──────────────────────────────────────────┘
                                  [↑] ← 플로팅 백투탑 (hero 섹션 외 항상)
                              [Modal] ← 방문예약 모달 (오픈 시 우측 슬라이드)
                              [Mobile Menu] ← 햄버거 클릭 시
```

스크롤 동작은 휠/터치로 섹션 단위 스냅 점프 (`page.tsx`의 `animateScrollTo`).

## 3. 섹션별 상세

### 3.1 ScrollHero

**컴포넌트 경로**: `src/components/sections/ScrollHero.tsx`

**Phase 진행** (자동 시퀀스, 클릭 X):
1. **boot** (300ms) — 검은 화면 settle
2. **line** (1900ms) — 한 글자씩 타이핑: "당신의 반려가족을 위한 모든 것, 베베펫입니다."
3. **copy** (700ms+500ms hold) — 타이핑 완료 후 커서 깜빡임 유지
4. **fadeOut** (650ms) — 텍스트 페이드아웃
5. **banner** (1100ms) — 라인이 가로로 전개 (28vh 높이 비디오 띠)
6. **full** (1000ms) — 비디오 띠가 위아래로 풀스크린 확장 → stage 1 진입

**Stage 단계** (사용자 휠 인터랙션으로 진행):
- **stage 1**: vis.mp4 풀스크린 + 우하단 hero 카피 (베베펫에서 시작되는 평생의 동행 / 평생 함께할 첫만남) + 모바일 RESERVATION 버튼 + 하단 스크롤 인디케이터
- **stage 2**: brand.mp4 패널이 우하단 22vw × 68vh 박스로 축소 + 좌측 ¾ 영역에 브랜드 카피 (동행, 가족이 되어, 평생을 함께하다 등)
- **stage 3**: brand.mp4가 다시 풀폭(왼쪽으로 전개) + 좌측 카피는 흰색으로 전환되어 비디오 위에 겹침

**우측 하단 Skip 버튼**: line 페이즈 동안만 표시. 클릭 시 인트로 즉시 스킵 → stage 1 풀스크린 비디오로 즉각 전환 (모핑 애니메이션 0ms).

### 3.2 CategoryIndex

**컴포넌트 경로**: `src/components/sections/CategoryIndex.tsx`

**구성**:
1. **배경 사진** (페이드 인) + **다크 그라데이션 오버레이** (하단 → 상단)
2. **CTA 안내 텍스트** (`베베펫이 제공하는 서비스를 확인해보세요.`) — hover 없을 때만
3. **하단 마퀴 라벨 5개**: ADOPT / HOSPITALITY / FOOD / PRODUCT / BEAUTY&SPA
4. **Hover Preview** (5종, 풀스크린 absolute overlay): 카테고리 hover 시 해당 카테고리 사진 + 라벨 + 캡션 + 본문 + CTA가 스태거 페이드인

**Hover preview 레이아웃** (50/50 split, visualSide 카테고리별 좌/우 교차):
```
┌──────────────┬──────────────┐
│              │  EYEBROW     │
│   VISUAL     │              │
│   (사진/카드) │  Caption (h2)│
│              │              │
│              │  Body 본문   │
│              │              │
│              │  [CTA]       │
└──────────────┴──────────────┘
```

특수 카테고리:
- **ADOPT**: visual = 3분할 카드 (대형견 / 소형견 / 고양이) + IG CTA
- **PRODUCT**: visual = 3×2 카드 그리드 (케어용품 / 의류 / 이동용품 / 위생용품 / 장난감 / 침구·매트)
- **PARTNERS** (FOOD): copy 영역 하단에 파트너 로고 마퀴
- **BEAUTY**: visual = 단일 사진 + "예약하기" CTA (방문예약 모달 오픈)

### 3.3 Contact (VISIT)

**컴포넌트 경로**: `src/components/sections/Contact.tsx` + `src/components/sections/LocationMap.tsx`

50/50 split:
- **좌**: 카카오맵 정적 이미지 + "베베펫 본사" 빨간 드롭핀 오버레이 + 어트리뷰션 strip
- **우**: VISIT 타이틀 (Jost) + lead 카피 + 위치/운영시간 dl

CategoryHoverPreview와 같은 X-axis (50% + 6vw padding + 28vw max-w) 정렬로 시선 이동 안정화.

### 3.4 Footer

**컴포넌트 경로**: `src/components/sections/Footer.tsx`

컴팩트 editorial 패턴, 두 영역만 노출:
1. **Brand block**: BEBE PET wordmark + 태그라인 (`footer.tagline`)
2. **Legal strip** (단일 가로선으로 구분):
   - 1행: 사업자정보 micro-print (대표 / 사업자등록번호 / 통신판매업신고번호) + 개인정보처리방침 링크
   - 2행: © copyright + Instagram 아이콘

매장 주소 · 전화 · 이메일 · 영업시간은 footer에 중복 표시하지 않으며 Visit 섹션(`#contact`)에서 일원화하여 노출합니다.

## 4. Header (모든 라우트 공통)

**컴포넌트 경로**: `src/components/Header.tsx`

```
┌─────────────────────────────────────────────────────────────┐
│ BEBE PET    분양 병원 사료/간식 용품 미용 공지사항 방문예약등록   KOR|ENG │
└─────────────────────────────────────────────────────────────┘
```

**테마 자동 전환** (`page.tsx`의 `headerTheme`):
- hero stage 1 → dark theme (흰 텍스트, vis.mp4 dark 배경)
- hero 외 stage / banner → light theme (검은 텍스트)
- index 섹션 + hover preview → preview 배경 톤 따라
- contact 섹션 → dark (어두운 사진 배경)

**모바일** (max-md): 햄버거 아이콘 → MobileMenu 열림.

## 5. RegisterModal (방문예약 모달)

**컴포넌트 경로**: `src/components/RegisterModal.tsx`

우측에서 슬라이드인되는 drawer (md:34vw, 모바일 전체 폭).

폼 필드:
1. **이름** (text, required)
2. **전화번호** (3 input split: 010 / 4자리 / 4자리)
3. **지역 (시/도)** (select, 17개 광역단체)
4. **상담목적** (radio chips × 4: 분양상담 / 병원진료 / 미용·스파 예약 / 기타)
5. **개인정보 수집 동의** (checkbox, required)

submit 시 `alert('방문 예약이 접수되었습니다. (데모)')` 표시 후 모달 닫힘.

> ⚠️ **백엔드 미연결**: 폼 제출은 데모. 실제 운영 시 이메일/Slack/DB 연동 필요. [04_운영배포매뉴얼.md](./04_운영배포매뉴얼.md) 참고.

## 6. 반응형 분기점

| 분기 | Tailwind | 적용 |
|---|---|---|
| 모바일 | `max-md` | < 768px |
| 데스크톱 | `md` | ≥ 768px |

폰트 크기는 대부분 `vw` 단위로 데스크톱은 자동 스케일. 모바일은 rem 기반 고정 사이즈 별도 지정.
