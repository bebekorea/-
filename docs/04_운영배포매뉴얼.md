# 04. 운영 · 배포 매뉴얼

본 문서는 빌드 · 배포 · 호스팅 · 모니터링 등 사이트 운영 전반의 절차를 정의합니다. 콘텐츠 수정에 대한 안내는 [05_콘텐츠운영매뉴얼.md](./05_콘텐츠운영매뉴얼.md)에 별도 정리되어 있습니다.

---

## 1. 로컬 개발 환경

### 1.1 사전 준비

- Node.js **v20 이상**
- npm v10+ (Node와 함께 설치됨)
- Git

### 1.2 의존성 설치

프로젝트 루트에서:

```bash
npm install
```

처음 한 번만 실행. node_modules 폴더에 설치됨 (~600MB).

### 1.3 개발 서버 실행

```bash
npm run dev
```

브라우저: http://localhost:3000

저장 시 자동 hot-reload (Next.js Turbopack).

### 1.4 빌드 확인

```bash
npm run build
```

성공 시 `.next/` 폴더에 정적 산출물 생성. 4개 라우트 (`/`, `/news`, `/privacy`, `/_not-found`) 모두 prerender.

## 2. 배포 — 추천 옵션 (Vercel)

### 2.1 추천 이유

- Next.js 제작사가 운영하는 호스팅
- 무료 티어 (월 100GB, 작은 사이트 충분)
- Git push 시 자동 배포 (CI/CD 내장)
- 자동 SSL (Let's Encrypt)
- 도메인 연결 1분

### 2.2 절차

1. https://vercel.com 가입 (GitHub 계정 권장)
2. **New Project** → 본 프로젝트 GitHub 리포 import
3. Framework: **Next.js** 자동 감지
4. Build command: `npm run build` (기본값)
5. Output directory: `.next` (기본값)
6. **Deploy** 클릭 → 1-2분 후 `bebepet-xxx.vercel.app` URL 발급
7. 도메인 구입 후 (가비아/후이즈 등): **Settings → Domains**에서 베베펫 도메인 연결 → SSL 자동 발급

### 2.3 환경변수 (필요 시)

현재는 환경변수 사용 X. 향후 추가 시:
- Vercel 대시보드 **Settings → Environment Variables**
- 클라이언트 노출은 `NEXT_PUBLIC_` prefix
- 시크릿(API 키 등)은 prefix 없이

### 2.4 자동 배포

- `main` 브랜치 push → 프로덕션 배포
- 다른 브랜치 push → 프리뷰 URL 자동 생성

## 3. 대안 호스팅

### 3.1 Netlify

- Vercel과 거의 동일. Next.js 지원 양호.
- 빌드: `npm run build`, publish: `.next`
- Next.js Runtime adapter 자동 설정

### 3.2 자체 서버 (VPS)

```bash
npm run build
npm run start  # 포트 3000에서 Next 서버 실행
```

`pm2` 또는 systemd로 데몬화. Nginx 리버스 프록시 + Let's Encrypt SSL 직접 구성.

### 3.3 정적 export (선택)

본 프로젝트는 모든 라우트가 정적 prerendered이므로 정적 export 가능:

```ts
// next.config.ts
export default { output: 'export' }
```

`npm run build` 후 `out/` 폴더를 GitHub Pages / S3 / Cloudflare Pages 등에 업로드.
단, `next/dynamic({ssr:false})`나 향후 서버 기능 추가 시 호환 안 될 수 있음.

## 4. 도메인 / SSL

### 4.1 도메인 구입

추천 등록기관:
- 가비아 (gabia.com) — 한국, .co.kr / .kr 도메인 강점
- 후이즈 (whois.co.kr) — 한국
- Cloudflare Registrar — 글로벌, 무이윤 도매가

### 4.2 DNS 연결 (Vercel 기준)

1. Vercel 프로젝트 → Settings → Domains → `bebepet.co.kr` 추가
2. 표시된 A/CNAME 레코드를 도메인 등록기관 DNS 콘솔에 입력
3. 5-30분 후 propagation 완료 → 자동 SSL 발급

### 4.3 SSL

Vercel/Netlify는 자동. 자체 서버는 Certbot:
```bash
sudo certbot --nginx -d bebepet.co.kr
```

## 5. 외부 서비스 의존

| 서비스 | 용도 | 무료 한도 | 라이선스 |
|---|---|---|---|
| **카카오맵 정적이미지 API** | Contact 섹션 지도 | 무제한 (계정 X) | 카카오맵 약관 (어트리뷰션 의무) |
| **Unsplash CDN** | ADOPT/PRODUCT 카테고리 사진 | 무제한 | Unsplash License (광고용 무료) |
| **Google Fonts (Noto Serif KR, Jost)** | 폰트 | 무제한 | OFL |
| **Pretendard CDN (jsdelivr)** | 한국어 본문 폰트 | 무제한 | OFL |

> ADOPT 및 PRODUCT 카테고리의 Unsplash placeholder 이미지는 운영 전 베베펫 실사진으로 교체할 것을 권장합니다 (광고 신뢰성 확보). 교체 절차는 [05_콘텐츠운영매뉴얼.md](./05_콘텐츠운영매뉴얼.md)에 정리되어 있습니다.

## 8. 트러블슈팅

| 증상 | 원인 / 조치 |
|---|---|
| 빌드 실패: `Can't resolve 'tailwindcss'` | `.next` 캐시 손상 → `rm -rf .next` → `npm run build` |
| 카카오맵 안 보임 | `staticmap.kakao.com` 도메인 차단 여부 확인 (방화벽) |
| 폰트 깨짐 | Pretendard CDN(jsdelivr) 접근 가능 여부 확인 |
| Vercel 빌드 OOM | Node 메모리 옵션: `NODE_OPTIONS=--max-old-space-size=4096 npm run build` |
| iframe 차단 | LocationMap은 정적 img 사용이라 iframe 아님 — 영향 없음 |
